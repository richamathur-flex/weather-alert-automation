What this system does (in simple terms)

Think of n8n as a digital conveyor belt. You place a trigger at the start (a timer, a webhook, an incoming email) and then chain action nodes together — each one does something: calls an API, filters data, sends a message, writes to a database. Data flows left-to-right through each node, getting transformed along the way.

Project 01 is the simplest version of this: a scheduled timer triggers an API call to fetch weather data, then formats and emails the result. Three nodes. One straight line. But it teaches every foundational concept you need.

How GitHub Connects to n8n

n8n and GitHub serve different but complementary roles. Here’s exactly how they work together:

CI/CD trigger✓ GitHub Trigger node✓ Can trigger n8n via webhooks

Three ways to connect them

Method 1 — Manual export (what we're doing now)
Export workflow JSON from n8n → commit to GitHub. Simple, reliable, works on any plan. Best for learning.
Method 2 — n8n GitHub Trigger node
n8n has a built-in GitHub node. It can listen for events like pushes, PRs, and issues — then take automated actions. Example: auto-label issues and send Slack notifications.
Method 3 — GitHub webhooks → n8n
Configure GitHub to send webhook events to your n8n instance. Gives you real-time triggers for any repo activity. Most flexible, requires a webhook URL from n8n.

End-to-End Walkthrough

Get OpenWeatherMap API Key

Go to openweathermap.org/api → Sign up (free) → Copy your API key from the dashboard. Free tier gives 1,000 calls/day.

Create New Workflow in n8n

Click “Create workflow” (red button, top right) → Name it Daily Weather Alert. You’ll see an empty canvas.

Add Schedule Trigger Node

Click “+” → Search “Schedule Trigger” → Add. Set Trigger Interval to “Days” and Hour to 8.

Add HTTP Request Node

Click “+” after the trigger → Search “HTTP Request”. Configure as GET request with query parameters for location, API key, and imperial units.

Config Method: GET URL: https://api.openweathermap.org/data/2.5/weather # Query Parameters: q = Little Elm,TX,US appid = YOUR_API_KEY_HERE units = imperial

Test the HTTP Node

Click “Test step”. You should see JSON with main.temp, main.humidity, weather[0].description. If you see data — your API connection works.

Add Gmail Node

Connect your Google account via credentials. Configure the email body using expressions:

Gmail To: your-email@gmail.com Subject: ☀ Weather Update — Little Elm, TX Body: Good morning! Condition: {{ $json.weather[0].description }} Temperature: {{ $json.main.temp }}°F Feels like: {{ $json.main.feels_like }}°F Humidity: {{ $json.main.humidity }}% Wind: {{ $json.wind.speed }} mph

Test Full Workflow

Click “Test Workflow” at the bottom. Check your inbox — you should receive the weather email within seconds.

Activate & Export

Toggle Active (top-right). Then three dots ⋮ → Download to export the workflow JSON. Commit to workflows/01-weather-alert.json.

📝 How I track errors
Every error encountered gets logged here with the exact error message, what caused it, and how I fixed it. This builds a personal troubleshooting reference.
401 Unauthorized — OpenWeatherMap APIResolved

Error: {"cod":401,"message":"Invalid API key"}

Cause: New API keys take up to 2 hours to activate after signup.

Fix: Wait 1-2 hours after creating your OpenWeatherMap account, then retry. Double-check for extra spaces when pasting the key.

Gmail Node — OAuth Scope ErrorResolved

Error: Insufficient Permission

Cause: Gmail OAuth scope did not include send permissions.

Fix: In Credentials → Gmail OAuth2 → re-authenticate and grant “Send email on your behalf” permission during the Google consent screen.

Expression Error — Cannot read property of undefinedResolved

Error: TypeError: Cannot read property 'description' of undefined

Cause: Used $json.weather.description instead of $json.weather[0].description. The weather field is an array.

Fix: Always check the output panel to see the exact JSON structure. Arrays need [0] indexing.

Expression Syntax Cheat Sheet
